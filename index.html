A vertical rhythm with a typographers scale & sensible line heights

Enlightened by Richard Rutter and Mark Boulton’s lectures and writings on vertical rhythm, I began to experiment with producing web pages that step to a beat. I was soon dismayed when I found using large font sizes for headings would need to be set on line heights in multiples of the base font size line height. The leading between headings would either be too tight or too loose. If I were to set the headings to a line height outside the multiples of the base line height, the proceeding paragraphs would fall out of sync. I seeked a dynamic way to use large font sizes, that have their own line height and yet allow restful placement of proceeding paragraphs back on the baseline.

With the development prowess of Andy MacDonald, I was able to describe to him my requirements as he tweaked and fettled together a script that works on the jQuery JavaScript library. The script is given the values of how many characters exist in a line of text at the given font sizes and then applies a class of lines-(x), where x is the number of lines. New margins may then be applied on a per line basis to elegantly place the proceeding paragraphs back in beat. I don’t forsee the headings that I use to ever span more than four lines, so the CSS statements do not grow unwieldly.

A dashboard to show how the script works

There is nothing like a good demonstration to replace a thousand words. In this working example I have provided a dashboard panel with buttons to turn off and on both the gridlines and the effects of the script upon the headings. Please take a moment to toggle the buttons on and off to witness exactly what the script achieves.

Automate adding a class to ampersands to make them look lovely

Dan Cederholm wrote some time a go about the importance of using the best available ampersands in heads and titles An ampersand is one of these ‘&’ if you weren’t sure. We are sing jQuery here to locate these beautiful characters, wrap them in a tag with a class if they are in a header, so we can style them to be even prettier.

You won’t find any widows mourning here

A typographic widow is the term used to refer to a single word on a line at the end of a paragraph. A widow is considered poor typography because it leaves too much white space between paragraphs or at the bottom of a page. Using Dave Cardwell’s excellent script – jqWidon’t – we are able to add a non-breaking space between the final and penultimate words.

The typographer’s scale adds measured increments, harmonious to one another

The scale was devised by some very clever typographer’s in the 16th Century, making it easier to set type. The scale used in this particular example page is ‘10px, 12px, 14px, 18px, 24px, 36px, 48px’

Featured item inside a box

The vertical measurements of this box are multiples of 18px and have top and bottom margins of 18px. It takes account of the 1px border too. The paragraphs proceeding this box rest back on the baseline, maintaining the vertical rhythm of the page.
